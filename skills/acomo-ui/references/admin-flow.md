# 管理者の操作フロー

ワークフロー管理（モデル・プロセスの監視）と、システム管理（ユーザー・グループ・ロール・認証・テナント設定）を扱う管理者向け案内。
権限の有無で見え方が変わるので、まず [`menu-and-permissions.md`](menu-and-permissions.md) を確認する。

## モデル管理 `/models`

- 一覧画面では、検索・絞り込みでモデルを探し、行を選ぶと詳細に入る。
- 新規作成は一覧右上の「**新規作成**」CTA から。
- モデルは「**下書き（編集中）**」と「**公開**」の 2 状態。**業務で使えるのは公開されたモデル** だけ。

### モデル詳細 `/models/:id`（編集ビュー）

- ページ上部に名前・説明・パンくず、ヘッダ帯にビュー切替リンクがある（編集と公開スナップショットを行き来できる）。
- 本文ではフロー図（ノード・遷移）、データスキーマ（フォーム項目）、ポリシー（誰がどの操作をできるか）を編集できる。
- 画面下部の Sticky アクションバーには、左側に **削除**（破壊的）、右側に **キャンセル／保存／公開** の流れで主要 CTA が並ぶ。
  - **保存する** … 下書きとして保存。業務にはまだ反映されない。
  - **公開する** … 編集内容を業務利用者にも使える状態にする（確認モーダルあり）。

### 公開スナップショット `/models/:id/published`

- 編集できない読み取り専用ビュー。**業務側で実際に動いている** モデルの姿を確認するときに使う。
- ヘッダのビュー切替リンクから「**編集に戻る**」で編集ビューへ戻れる。

## プロセス監視 `/processes`

- テナント全体のプロセスを横断的に見る画面。検索・状態フィルタで対象を絞り、行を選んで詳細 `/processes/:id` を確認する。
- 管理者ビューでは、業務利用者ビューと違い、フォーム入力ではなく **進行状況・履歴・データの読み取り** が主目的。
- 必要であれば管理者用のアクション（履歴からの復元など）が画面 CTA に従って出る。

## ユーザー管理 `/users`

- 一覧から検索・絞り込みでユーザーを探す。
- 「**新規作成**」でユーザーを招待する（メール・初期ロールなどを設定）。
- 詳細画面はタブで「基本」「セキュリティ」などに分かれ、ロール付与・無効化などができる。

## グループ管理 `/groups`

- グループは **ツリー構造**。親グループに付けたロールは子に効く設計に揃えるのが基本。
- 一覧でツリーを開閉しながら確認し、「**新規作成**」で配下グループを追加する。
- 個別のユーザーに直接ロールを付けるよりも、**グループ単位でロールを集約** すると運用が楽。

## ロール管理 `/roles`

- ロールは「何をして良いか（権限の集合）」を 1 つにまとめたもの。
- 「**新規作成**」でロールを足し、対象（モデル、プロセス、ユーザー、ロール、グループ、認証、設定 など）ごとに、参照のみ／編集／管理者の段階を選んでチェックする。
- 名前は **業務上の役割が伝わる名前** を勧める（例: 「経理承認者」「モデル編集者」）。

## 認証設定 `/authConfig`

- タブで以下を扱う。
  - **認証プロバイダー** … メール／OAuth／OIDC など、ログイン方法の有効化と設定。
  - **CORS** … 別オリジンからの API 呼び出しを許可するドメイン。
  - **アクセストークン** … API 連携用の長期トークンの発行・失効。
- 認証まわりの変更は **既存ユーザーのログインに影響** することがあるため、業務時間外の変更を勧める案内をしてよい（具体的な時間は推測しない）。

## テナント設定 `/settings`

- テナント単位の各種設定。Slack 連携などの外部統合もここから扱う。
- 利用者全員に影響する変更が多いので、保存前に変更点をユーザーに確認するよう促す。

## 管理者向けの問い合わせ対応の型

- **「○○ メニューがメンバーに見えない」**
  → そのメンバーのロール・グループに、対応するカテゴリ（モデル管理／ユーザー管理 など）の権限が付いているかを確認するよう案内する。
- **「モデルを公開したのに業務側で使えない」**
  → 利用者側の「プロセスを開始する」一覧（`/my/models`）に、開始権限のあるユーザーで入って確認するよう促す。モデルは公開済みでも、開始権限が付与されていなければ一覧に出ない。
- **「ユーザーを無効化したい」**
  → ユーザー詳細から無効化操作。具体的なボタン名は画面の CTA に従う、と添える（実装で変わり得るため）。

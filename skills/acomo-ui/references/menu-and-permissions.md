# サイドメニューと権限

acomo のサイドメニュー（左に開くドロワー）は、**ログイン中のユーザーが持つ権限** によって自動的にフィルタされる。
権限がないカテゴリ・項目はメニューに出ない。「画面を開く URL を直打ちする」のではなく、まずはメニューの並びで自分の見え方を確認するのが基本。

## カテゴリと項目（テナントにログイン中のとき）

| カテゴリ | 項目（メニュー名） | 主な用途 | 表示に必要な権限 |
| --- | --- | --- | --- |
| 自分のワーク | ダッシュボード | 自分の待機タスク・進行中／過去のプロセス確認 | 全ログインユーザーに表示 |
| ワークフロー管理 | モデル | ワークフローモデルの一覧・編集・公開 | モデル管理権限 |
| ワークフロー管理 | プロセス | テナント全体のプロセス一覧・状況把握 | プロセス管理権限 |
| システム管理 | ユーザー | 利用者の招待・無効化・ロール付与 | ユーザー管理権限 |
| システム管理 | グループ | グループ（ツリー）の作成・ロール集約 | グループ管理権限 |
| システム管理 | ロール | ロールの新規作成・権限の組み合わせ設定 | ロール管理権限 |
| システム管理 | 認証 | 認証プロバイダー・CORS・アクセストークン | 認証設定権限 |
| システム管理 | 設定 | テナント全体の設定（外部連携など） | テナント設定権限 |
| 開発者ツール | API / OpenAPI 仕様書 / AI アシスタント | API・自動化向け | 開発者権限 |
| サポート | サポート | 問い合わせ送信 | 全ログインユーザーに表示 |

> 用語の意味は [`glossary.md`](glossary.md) を参照する。

## アカウント直下（テナントを未選択のとき）

テナントを選ぶ前のアカウント・組織コンテキストでは、メニューがアカウント管理向けに切り替わる。

- アカウントの管理: 「テナント」一覧、「サブスクリプション」
- ログインしたあと、画面右上のテナント切り替えメニューから業務テナントへ移動する

## 「目的のメニューが見つからない」ときの案内テンプレ

利用者から「○○ メニューが出てこない／○○ 画面が見当たらない」と相談されたら、次の順で確認を案内する。

1. **正しいテナントにログインしているか** を確認する（画面右上のテナント切り替えで業務テナントに移っているか）。
2. その項目は **権限が必要** で、自分のロール／グループに割り当たっていないと出ない可能性がある。
3. 自分の権限を確認するには、ダッシュボード `/my` から「自分のワーク」配下の **権限の確認画面（`/my/actions`）** を見る。
4. 必要なら、ロールを付けてくれる **テナントの管理者へ依頼** するよう促す。

> 推測でロール名・管理者名を埋めない。「あなたのテナント管理者」「ユーザー管理権限を持っている方」のような一般的な表現に留める。

## 「メニューはあるのにボタンが押せない」とき

メニューが表示されても、画面内の **編集・公開・削除など強い操作だけが押せない** ケースがある。これは「参照」までは許可されているが「編集」「管理」までは許可されていない、という権限の段階差によるもの。同じく管理者へロールの確認を依頼するよう促す。

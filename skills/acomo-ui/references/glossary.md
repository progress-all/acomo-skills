# 用語集（acomo Web UI）

acomo の画面案内で頻出する用語を、**画面上の意味** に絞って整理する。実装名・API 物理名は併記しない（必要なら開発者向けモードへ切り替えを促す）。

## ワークフロー周りのコア用語

### モデル
- ワークフローの **定義（ひな型）**。フロー図、入力フォームの項目（データ）、誰がどの操作をできるかのルールがまとまっている。
- 「下書き（編集中）」と「公開」の 2 つの状態がある。**業務で使えるのは公開されたモデル** だけ。
- 編集権限を持つのはワークフロー管理者・モデル管理者など。

### プロセス
- モデルから **1 件開始された実行インスタンス**。1 件の申請書・依頼・チケット相当。
- データ（フォームの入力値）と進行状況（今どのノードにいるか）を持つ。
- 業務利用者は「自分のワーク」配下、管理者はテナント横断の「プロセス」一覧から確認できる。

### ノード
- ワークフロー図上の 1 つのステップ。「申請」「上長承認」「最終承認」など。
- ノードに **トークン** が乗っている間、そのノードに割り当てられた利用者がアクションを実行できる。

### アクション
- ノード上で実行できる操作の種別。画面上のボタン文言は命令形で表示される。
  - **提出する** … 申請者が下書きを完成させて次のノードへ送る。
  - **承認する** … 提出されたものを認め、次へ進める。
  - **却下する** … 認めず、前のノードに差し戻す。
  - **保存する** … 入力フォームだけを保存する。**ワークフローは進行しない**。
  - **取り戻す** … 自分が直前に提出したノードへ戻す（やり直し）。
  - **開始する** … 公開モデルから新しいプロセスを開始する。
- 進行が変わるアクション（提出・承認・却下・取り戻し）は、**実行前に確認モーダル** が挟まる。

### タスク
- 「**自分が今、行動できるノード**」のこと。ダッシュボード `/my` の「今対応できる」タブに並ぶ件数がこれにあたる。
- アクションを実行する権限がある／そのノードにトークンが乗っている、両方が満たされたものだけが見える。

### トークン
- ワークフロー図上の **「現在地」マーカー**。ノードに乗っているトークンが、そのノードのアクションを可能にする。
- 並列分岐があれば、同じプロセスの中で複数のノードに同時にトークンが乗ることがある。

## 組織・権限まわり

### テナント
- ログイン後に選ぶ **組織コンテキスト**。同じユーザーが複数テナントに所属していることもあり、テナントを切り替えるとサイドメニューや見えるデータが変わる。
- テナントはアカウント直下の「テナント」一覧（`/account/tenants`）から選び直せる。

### ユーザー
- ログインアカウント。テナントごとに付与されたロール／グループに応じて、できることが変わる。

### ロール
- 「何をして良いか」のまとまり。**権限はロールに集約する** のが基本。
- ロールには、対象（モデル、プロセス、ユーザー、ロール、グループ、認証、設定 など）ごとに、参照のみ／編集／管理者の権限を設定する。

### グループ
- ユーザーを束ねる入れ物。**ツリー構造** に組める。
- グループにロールを付けると、そのグループに属するユーザー全員にロールが行きわたる。

### 権限の付与関係（ざっくり）

```
権限 ─（割り当て）→ ロール ─（付与）→ ユーザー / グループ ─（所属）→ ユーザー
```

「自分にどの権限があるか」は、自分が属するロール・グループの組み合わせで決まる。

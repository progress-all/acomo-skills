# 業務利用者の操作フロー

「自分のワーク」配下（`/my`）を中心に、業務利用者がよく行う操作を画面の動きでまとめる。
用語は [`glossary.md`](glossary.md)、画面と URL の対応は [`screens.md`](screens.md)、権限の見え方は [`menu-and-permissions.md`](menu-and-permissions.md) を参照する。

## ダッシュボード `/my` の見方

- ページタイトル: 「自分のワーク」
- 主要 CTA: ヘッダ右上の「**プロセスを開始する**」ボタン → `/my/models` へ移動
- タブは 3 つ。各タブには件数バッジが付く。
  - **今対応できる** … 自分が次にアクションを起こせるプロセスの一覧（タスク）。最初に見るべきタブ。
  - **過去対応した** … 自分が過去に何らかのアクションを行ったプロセス。
  - **すべて** … 自分が関わっている範囲のすべて。
- 一覧から行を選ぶとプロセス詳細 `/my/processes/:id` に入る。

## プロセスを開始する

1. ダッシュボード `/my` の「**プロセスを開始する**」を押す。
2. `/my/models`（自分が開始できるモデルの一覧）が開く。利用したいモデルを選ぶ。
3. `/my/models/:id` でモデル内容を確認し、「**プロセスを開始する**」を押す。
4. 開始されたプロセスの詳細 `/my/processes/:id` に着地する。最初のノードのフォーム入力から始める。

> モデル一覧に目当てのモデルが見当たらないときは、そのモデルがまだ公開されていないか、自分にプロセス開始の権限が付いていない可能性がある。

## 自分のプロセス詳細 `/my/processes/:id`

ページ上部にプロセス名・パンくず（自分のワーク › プロセス名）と、状態を示すヘッダ帯（進行状況・ビュー切替）が並ぶ。本文は入力フォーム（プロセスデータ）が中心で、進行状況や履歴は同ページ内の補助タブ／表示で確認できる。

### 画面下部の操作バー（Sticky）

画面下に張り付くアクションバーには、現在のノードでできる操作だけが出る。**ボタンはすべて命令形** で、ワークフローを進める操作（提出・承認・却下・取り戻し）は **押した直後に確認モーダル** が挟まる。

| ボタン | 何が起きるか | 確認モーダル |
| --- | --- | --- |
| **キャンセル** | 入力を破棄してダッシュボード `/my` に戻る | なし |
| **保存する** | 入力フォームのみ保存する。ワークフローは **進行しない** | なし |
| **却下する** | 提出されたものを認めず、前のノードへ差し戻す | あり |
| **承認する** | 提出されたものを認めて次のノードへ進める | あり |
| **提出する** | 申請者が下書きを完成させ、次のノードへ送る | あり |
| **取り戻す** | 自分が直前に提出したノードに戻してやり直す。複数候補があれば、ノード選択モーダルで対象を選ぶ | あり（取り戻し対象の確認） |

実行が成功すると画面右下に「正常に処理しました。」と通知され、ダッシュボード `/my` に戻る。失敗するとエラー通知が出るが、データの推測案内はしない（再現手順だけを丁寧に）。

### 「対応できるノードがありません」と表示されるとき

- そのプロセスは別の利用者の対応待ち（自分のターンではない）。
- 取り戻しだけが可能なケースもあり、その場合は Sticky バーに「取り戻す」だけが出る。

## よくある問い合わせと案内の型

- **「自分の権限を確認したい」**
  → 「自分のワーク」配下の `/my/actions`（権限の確認画面）で、現在のテナントで自分に付与された権限が見られる。
- **「却下されたプロセスをもう一度提出したい」**
  → 却下されたものは前ノードに戻っているので、ダッシュボード `/my` の「今対応できる」に戻ってきているはず。そこからプロセスを開き、フォームを直して「**提出する**」を押す。
- **「うっかり提出してしまった」**
  → 直前に提出したノードに戻れる場合は、Sticky バーに「**取り戻す**」が出ている。出ていないときは、次のノード担当者に却下してもらう必要がある。
- **「モデル一覧に目的のモデルがない」**
  → モデルが公開されていないか、自分にプロセス開始の権限がない可能性。テナント管理者・モデル管理者へ依頼する。

# 主要画面と URL（要約）

権限がないメニューはサイドバーに表示されない。以下は代表的な業務利用者・管理者向けパス。各行は **画面名（ページタイトル）／主要 CTA／タブ構成** の順で 1〜2 行に整理している。

## 自分のワーク（業務利用者）

| 目的 | URL（目安） | ページタイトル / 主な操作 |
| --- | --- | --- |
| ダッシュボード・進行中プロセス | `/my` | 「自分のワーク」。CTA: 「プロセスを開始する」。タブ: 今対応できる／過去対応した／すべて |
| プロセスを開始（利用可能モデル） | `/my/models` | 自分が開始できるモデル一覧。モデルを選ぶと詳細へ |
| モデル内容の確認（読み取り） | `/my/models/:id` | 読み取り専用。CTA: 「プロセスを開始する」 |
| 自分のプロセス詳細 | `/my/processes/:id` | フォーム入力＋下部 Sticky の操作（保存／提出／承認／却下／取り戻し） |
| 自分に付与された権限の一覧 | `/my/actions` | 自分のロール・グループから集約された権限の確認用 |

## ワークフロー管理（設計・運用）

| 目的 | URL（目安） | ページタイトル / 主な操作 |
| --- | --- | --- |
| モデル一覧・検索・新規 | `/models` | CTA: 「新規作成」。一覧から行を選んで編集へ |
| モデル編集 | `/models/:id` | フロー図／データスキーマ／ポリシーを編集。Sticky: 削除（左）／キャンセル・保存・公開（右） |
| モデル公開スナップショット | `/models/:id/published` | 業務側で動いているモデルの読み取り専用ビュー |
| プロセス一覧（テナント横断） | `/processes` | テナント全体の状況把握。検索・状態フィルタ |
| プロセス詳細（管理者視点） | `/processes/:id` | 進行状況・履歴・データの読み取りが主 |

## システム管理

| 目的 | URL（目安） | ページタイトル / 主な操作 |
| --- | --- | --- |
| ユーザー | `/users` | CTA: 「新規作成」。一覧／詳細（基本・セキュリティ） |
| グループ | `/groups` | ツリー表示。配下グループの作成・ロール集約 |
| ロール | `/roles` | CTA: 「新規作成」。対象ごとに参照／編集／管理を設定 |
| 認証 | `/authConfig` | タブ: 認証プロバイダー／CORS／アクセストークン |
| 設定 | `/settings` | テナント全体の設定（外部連携など） |

## 補助

| 目的 | URL（目安） | 備考 |
| --- | --- | --- |
| サポート問い合わせ | `/support` | 全ログインユーザー |
| プロフィール | `/profile` | 自分のアカウント情報 |
| アカウント・組織・テナント（公開コンテキスト） | `/account` 配下 | テナント未選択時のメニュー |

## 開発者ツール

| 目的 | URL（目安） | 備考 |
| --- | --- | --- |
| API ドキュメント | `/development/apis` | 開発者権限 |
| OpenAPI 仕様書 | `/api` | 外部リンク |
| AI アシスタント（汎用チャット） | `/ai/agent` | 利用可否はサーバ構成に依存 |
| ワークフローモデリングAIエージェント | `/ai/agent/workflow-model` | 汎用の「AI アシスタント」とは別。チャットから説明を始め、モデル案やデータ項目・ポリシーを対話で進める。モデル一覧の「ワークフローモデリングAIエージェントで作成」または AI アシスタント画面の専用カードから遷移 |

## 補足

- 典型的な業務フロー（申請・承認・却下・取り戻し）の操作詳細は [`business-user-flow.md`](business-user-flow.md) を参照する。
- 管理者の操作詳細は [`admin-flow.md`](admin-flow.md) を参照する。
- 詳細な UI ラベルやタブ構成はリリースで変わり得るため、**メニュー名と URL** を手がかりに案内し、画面上の同名の項目を探すよう促す。

# acomo 画面操作ガイド（プロダクト UI）

このスキルは **acomo Web UI** の使い方を案内する。回答は次を優先する。

- 左サイドメニューからのたどり方、画面タイトル、主要なボタン・タブの意味
- 権限によってメニュー・操作が出ない場合の説明（一般論）
- ワークフロー利用者向け: ダッシュボード、モデルからのプロセス開始、自分のプロセスでの申請・承認・却下・取り戻しの流れ
- ワークフロー管理者・システム管理者向け: モデル編集と公開、ユーザー・グループ・ロール・認証設定

## 質問パターン → 参照ファイル（エージェント用）

ユーザーの質問が次のどれに当たるかを判断し、該当ファイルを必ず確認してから答える。憶測で画面名・ボタン名・タブ名を作らないこと。

次表は **エージェントがどの資料を読むか** のマッピングのみ。**ユーザー向け説明で資料名やパスを出さない作法**はリポジトリ共通（`AGENTS.md`「ユーザー向け回答（全スキル共通）」）を正とする。

| 質問の種類 | 参照ファイル |
| --- | --- |
| 用語の意味（モデル／プロセス／アクション／タスク／トークン／テナント／ロール／グループ） | [`references/glossary.md`](references/glossary.md) |
| サイドメニューの並び、メニューが見えない・操作が押せない原因 | [`references/menu-and-permissions.md`](references/menu-and-permissions.md) |
| 業務利用者の操作（ダッシュボードの見方、プロセス開始、申請・承認・却下・取り戻し・保存） | [`references/business-user-flow.md`](references/business-user-flow.md) |
| 管理者の操作（モデル編集と公開、ユーザー／グループ／ロール／認証設定／テナント設定） | [`references/admin-flow.md`](references/admin-flow.md) |
| URL から画面の役割を逆引き、画面と CTA・タブの早見 | [`references/screens.md`](references/screens.md) |

複数のカテゴリにまたがる質問では、それぞれの参照ファイルを順に確認すること（例: 「モデルとプロセスの違い」は用語＋業務者フロー、「自分の権限を確認したい」はメニュー権限＋業務者フロー）。

## 回答の作り方

- ユーザーへの開示の作法は **`AGENTS.md`「ユーザー向け回答（全スキル共通）」** に従う（本スキルは Web UI の手順・用語に集中する）。
- 操作手順は **番号付き** で書く。画面上で探すべき UI ラベルは「」で括る（例: ヘッダ右上の「プロセスを開始する」）。
- 申請・承認・却下・取り戻しなど **ワークフロー進行系の操作** は、押したあとに **確認モーダル** が挟まることを必ず添える。
- 目的の画面・操作が見当たらないという相談には、まず **権限不足** の可能性を案内し、必要に応じて管理者への依頼を促す。
- 「保存する」と「提出する」の違いのように、**ワークフローを進める／進めない** の境目は明示する。
- リリースで UI ラベル・タブ名は変わり得るため、メニュー名と URL を手がかりに「画面上で同名の項目を探してください」と添える。

## やらないこと

- `@acomo/cli` のコマンド例、REST のパス・ボディ、トークン取得手順など **開発者向け API/CLI** の具体手順
- acomo 本体の内部実装やリポジトリ構成の説明
- ユーザー固有のデータ（特定プロセス ID の状態、特定ユーザーのロール構成など）の推測

これらが必要なら、**開発者向け（API/CLI）モード**への切り替えをユーザーに促す。
